Task Management System

Complete Documentation & Technical Reference

📋 Project Overview

A full-stack task management application with Role-Based Access Control (RBAC) that enables organizations to manage tasks efficiently with different permission levels for Admin, Manager, and User roles.

🔐

JWT Authentication

Secure token-based authentication with 7-day expiration

👥

RBAC System

Three roles with granular permissions

📝

Task Management

Full CRUD with filtering and assignment

🎨

Modern UI

React with Tailwind CSS responsive design

🎯 Key Capabilities

  • User Registration & Login - Secure authentication with password hashing
  • Task CRUD Operations - Create, read, update, delete tasks
  • Role-based Permissions - Different access levels for each role
  • Task Assignment - Managers can assign tasks to team members
  • Admin Impersonation - Admin can login as any user for testing
  • Task Filtering - Filter by status, priority, and assignee

SQLite

Database

Why SQLite?

  • Zero Configuration: No server setup, runs embedded in the application
  • Serverless: Single file database - easy to backup, move, and version control
  • ACID Compliant: Full transaction support with rollback capabilities
  • Cross-Platform: Works on Windows, Linux, macOS without changes
  • Perfect for Development: No external dependencies, instant setup

⚠️ When to Use PostgreSQL/MySQL Instead
  • High-write concurrent applications (>100 writes/second)
  • Multiple servers need to access the same database
  • Database size exceeds 100GB
  • Need advanced features like stored procedures

🔐 JWT Authentication

How JSON Web Tokens Secure Your Application

What is JWT?

JSON Web Token (JWT) is an open standard (RFC 7519) for securely transmitting information between parties as a JSON object. This information is digitally signed, so it can be verified and trusted.

JWT Structure

Header

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9 
{
  "alg": "HS256",
  "typ": "JWT"
}

Specifies the signing algorithm (HS256) and token type

.

Payload

eyJ1c2VySWQiOjEsImVtYWlsIjoiYWRtaW5AYWRtaW4uY29tIiwicm9sZUlkIjoxLCJpYXQiOjE3MTIwMDAwMDAsImV4cCI6MTcxMjYwNDgwMH0 
{
  "userId": 1,
  "email": "admin@admin.com",
  "roleId": 1,
  "iat": 1712000000,
  "exp": 1712604800
}

Contains user data and token metadata

.

Signature

SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c 
HMACSHA256(
  base64UrlEncode(header) + "." +
  base64UrlEncode(payload),
  secret
)

Ensures token hasn't been tampered with

Token Lifecycle

🔑

1. Token Generation

When user logs in successfully, server creates a JWT with user data

const token = jwt.sign(
  { userId: user.id, email: user.email, roleId: user.roleId },
  JWT_SECRET,
  { expiresIn: '7d' }  // Token valid for 7 days
);
💾

2. Token Storage

Frontend stores token in localStorage for persistence

localStorage.setItem('token', response.data.token);
localStorage.setItem('user', JSON.stringify(response.data.user));
📤

3. Token Transmission

Every API request includes token in Authorization header

// Axios interceptor automatically adds token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
✅

4. Token Verification

Server validates token on every protected request

const decoded = jwt.verify(token, JWT_SECRET) as TokenPayload;
// decoded = { userId: 1, email: "...", roleId: 1, iat: ..., exp: ... }

// Fetch fresh user data from database
const user = await prisma.user.findUnique({
  where: { id: decoded.userId },
  include: { role: { include: { permissions: true } } }
});
⏰

5. Token Expiration

After 7 days, token becomes invalid and user must re-login

// Server returns 401 if token expired
if (error instanceof jwt.TokenExpiredError) {
  res.status(401).json({ message: 'Token expired' });
}

// Frontend redirects to login on 401
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
  }
);

Token Configuration

Setting Value Description
JWT_SECRET Environment Variable Secret key for signing tokens. Never commit to git!
JWT_EXPIRES_IN 7 days (7d) Token validity period. User must re-login after expiration.
Algorithm HS256 HMAC with SHA-256 - symmetric signing algorithm

⏱️ Token Expiration Options

60 = 60 seconds
'2 days' = 2 days
'10h' = 10 hours
'7d' = 7 days ✓
'1w' = 1 week
'1y' = 1 year

Security Best Practices

✅

We Do

  • Store JWT_SECRET in environment variables
  • Set reasonable expiration (7 days)
  • Validate token on every request
  • Hash passwords with bcrypt (10 rounds)
  • Clear token on logout
  • Validate token on app reload
❌

Never Do

  • Store JWT_SECRET in code
  • Set token to never expire
  • Store sensitive data in JWT payload
  • Trust token without verification
  • Store tokens in cookies without httpOnly
  • Log full JWT tokens

Prisma Schema

// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")  // file:./dev.db
}

model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  username  String   @unique
  password  String
  roleId    Int      @map("role_id")
  role      Role     @relation(fields: [roleId], references: [id])
  tasks     Task[]   @relation("CreatedTasks")
  assigned  Task[]   @relation("AssignedTasks")
  createdAt DateTime @default(now()) @map("created_at")
  updatedAt DateTime @updatedAt @map("updated_at")

  @@map("users")
}

model Role {
  id          Int              @id @default(autoincrement())
  name        String           @unique
  description String?
  users       User[]
  permissions RolePermission[]
  createdAt   DateTime         @default(now()) @map("created_at")

  @@map("roles")
}

model Permission {
  id          Int              @id @default(autoincrement())
  name        String           @unique
  description String?
  roles       RolePermission[]

  @@map("permissions")
}

model RolePermission {
  roleId       Int        @map("role_id")
  permissionId Int        @map("permission_id")
  role         Role       @relation(fields: [roleId], references: [id], onDelete: Cascade)
  permission   Permission @relation(fields: [permissionId], references: [id], onDelete: Cascade)

  @@id([roleId, permissionId])
  @@map("role_permissions")
}

model Task {
  id          Int       @id @default(autoincrement())
  title       String
  description String?
  status      String    @default("TODO")
  priority    String    @default("MEDIUM")
  userId      Int       @map("user_id")
  user        User      @relation("CreatedTasks", fields: [userId], references: [id], onDelete: Cascade)
  assignedTo  Int?      @map("assigned_to")
  assignee    User?     @relation("AssignedTasks", fields: [assignedTo], references: [id], onDelete: SetNull)
  dueDate     DateTime? @map("due_date")
  createdAt   DateTime  @default(now()) @map("created_at")
  updatedAt   DateTime  @updatedAt @map("updated_at")

  @@map("tasks")
}

Permission Middleware Code

// middleware/permission.ts

export const hasPermission = (...requiredPermissions: string[]) => {
  return (req: AuthRequest, res: Response, next: NextFunction): void => {
    if (!req.user) {
      res.status(401).json({ message: 'Not authenticated' });
      return;
    }

    const userPermissions = req.user.role.permissions;
    
    // Check if user has ANY of the required permissions (OR logic)
    const hasRequired = requiredPermissions.some(perm => 
      userPermissions.includes(perm)
    );

    if (!hasRequired) {
      res.status(403).json({ 
        message: 'Access denied. Insufficient permissions.'
      });
      return;
    }

    next();
  };
};

// Usage in routes:
router.get('/tasks', hasPermission('tasks:read:own', 'tasks:read:all'), getTasks);
router.post('/users', hasPermission('users:create'), createUser);
