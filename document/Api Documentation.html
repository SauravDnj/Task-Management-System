Overview

Introduction

The Task Management System API is a RESTful API that returns JSON. All requests must include Content-Type: application/json in the request headers for endpoints that accept a body.

Request Format

All POST / PUT / PATCH requests must send a JSON body. All protected routes require the Authorization: Bearer <token> header. The token is obtained from the Login endpoint.

Example — every authenticated request
GET http://localhost:5001/api/tasks Content-Type: application/json Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Overview

Authentication

This API uses JWT (JSON Web Tokens) for authentication. After logging in, include the token in all subsequent requests using the Bearer scheme.

Token payload structure (decoded)
{ "userId": 4, "email": "saurav@example.com", "roleId": 1, "iat": 1775074963, "exp": 1775679763 }

Error Handling

All errors return a consistent JSON structure. HTTP status codes indicate the type of error.

Status Code Meaning Common Cause
200 OKSuccessRequest completed successfully
201 CreatedResource createdPOST request succeeded
400 Bad RequestValidation errorMissing or invalid fields in request body
401 UnauthorizedAuth requiredMissing or expired JWT token
403 ForbiddenAccess deniedRole does not have permission
404 Not FoundResource missingTask or user ID does not exist
500 Server ErrorInternal errorUnexpected server failure
Error response shape
{ "message": "Invalid credentials" }
